# Governance

## Decision Making

### Major Decisions
- Require approval from 2+ core team members
- Document rationale
- Consider community feedback

### Minor Changes
- Can be made by single maintainer
- Should be reversible

## Release Process

1. Feature freeze
2. Testing phase (1 week)
3. Beta release
4. Community feedback (1 week)
5. Stable release

## Security Issues

- Report privately
- Fix within 48 hours
- Release security patch
- Public disclosure after 7 days
